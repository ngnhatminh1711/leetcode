Big-O & Complexity Analysis

"Bạn không thể chọn đúng thuật toán nếu không biết nó tốn bao nhiêu."
🟢 BeginnerPrereq cho mọi module
📊 Bảng Big-O
Notation Tên n=100 n=10⁵ Ví dụ
O(1) Constant 1 1 HashMap.get()
O(log n) Logarithmic 7 17 Binary search
O(n) Linear 100 100,000 Array scan
O(n log n) Linearithmic 700 1,700,000 Arrays.sort()
O(n²) Quadratic 10,000 10¹⁰ ⚠️ Nested loops
O(2ⁿ) Exponential ☠️ ☠️ Naive recursion
🎯 Interview target: hầu hết Medium chấp nhận O(n log n) hoặc tốt hơn. O(n²) thường pass với n ≤ 1000.
📏 Quy tắc & Traps thường gặp

3 Quy tắc cơ bản

// Quy tắc 1: Drop constants
O(2n) → O(n),  O(500) → O(1)

// Quy tắc 2: Drop non-dominant terms
O(n² + n) → O(n²),  O(n + log n) → O(n)

// Quy tắc 3: Loop độc lập = cộng; Lồng = nhân
for i in n: ...     // O(n)
for j in m: ...     // + O(m) → tổng O(n+m)

for i in n:
  for j in n: ...   // × O(n) → O(n²)

Common Traps

String += trong loop: mỗi phép + tạo new String → O(n²). Dùng StringBuilder.append() → O(n)
List.contains() thay Set.contains(): List O(n), Set O(1). Gọi trong loop → List tạo O(n²), Set giữ O(n)
Quên recursion stack space: DFS trên tree → O(height). Balanced = O(log n), Skewed = O(n)

Phân tích Time và Space complexity của các đoạn code sau: 

// Đoạn A
public int mystery(int n) {
    int count = 0;
    for (int i = n; i > 0; i /= 2)
        count++;
    return count;
}

// Đoạn B
public boolean hasDuplicate(int[] arr) {
    for (int i = 0; i < arr.length; i++)
        for (int j = i + 1; j < arr.length; j++)
            if (arr[i] == arr[j]) return true;
    return false;
}

// Đoạn C
public List<List<Integer>> subsets(int[] nums) {
    List<List<Integer>> res = new ArrayList<>();
    backtrack(nums, 0, new ArrayList<>(), res);
    return res;
}
// backtrack generates all subsets
  1. Đoạn A: Quan sát i /= 2 — mỗi bước i giảm đi một nửa. Hỏi: bao nhiêu lần bạn có thể chia đôi n trước khi đến 0?
  2. Đoạn B: Đếm số lần chạy của inner loop với mỗi giá trị của outer loop: i=0 → n-1 lần, i=1 → n-2 lần... Tổng là bao nhiêu?
  3. Đoạn C: Mỗi phần tử có 2 lựa chọn: có hoặc không có trong subset. n phần tử → bao nhiêu subset? Space tính cả output.
Đoạn A
Time: O(log n) — i bị chia đôi mỗi vòng lặp (1, 1/2, 1/4...). Sau log₂(n) bước thì i = 0.
Space: O(1) — chỉ dùng biến count và i.
Đoạn B
Time: O(n²) — outer loop n lần, inner loop ~n/2 lần average. Tổng = n*(n-1)/2 ≈ n²/2 → drop constant → O(n²).
Space: O(1) — in-place, không allocate thêm.
Đoạn C
Time: O(n × 2ⁿ) — có 2ⁿ subset, mỗi subset tốn O(n) để copy vào result.
Space: O(n × 2ⁿ) — output chứa 2ⁿ subset, mỗi subset dài tối đa n phần tử. Call stack thêm O(n).
Key takeaway: log n xảy ra khi input bị chia nhỏ theo tỉ lệ. 2ⁿ xảy ra khi mỗi phần tử có 2 lựa chọn độc lập.

✅ Đáp án
Mục tiêu
Với 3 solution của bài Two Sum bên dưới, hãy phân tích Time và Space complexity của mỗi cách. Cách nào tốt nhất? Tại sao? 
// Solution 1: Brute Force
for (int i = 0; i < n; i++)
  for (int j = i+1; j < n; j++)
    if (nums[i] + nums[j] == target) return {i, j};

// Solution 2: Sort + Two Pointers
Arrays.sort(nums);
int l = 0, r = n-1;
while (l < r) {
  int s = nums[l] + nums[r];
  if (s == target) return ...;
  else if (s < target) l++;
  else r--;
}

// Solution 3: HashMap
Map<Integer,Integer> map = new HashMap<>();
for (int i = 0; i < n; i++) {
  if (map.containsKey(target - nums[i]))
    return {map.get(target-nums[i]), i};
  map.put(nums[i], i);
}
  1. Solution 1: Đếm số lần inner loop chạy với outer loop n lần.
  2. Solution 2: Sorting tốn bao nhiêu? Sau khi sort, two pointers chạy bao nhiêu bước? Cộng lại. Lưu ý: sort có thể mất thông tin index gốc!
  3. Solution 3: HashMap.get() và HashMap.put() mỗi cái tốn bao nhiêu? Space: map có tối đa bao nhiêu entries?
Solution 1 — Brute Force
Time: O(n²) | Space: O(1)
Không tốt. n=10⁵ → 10¹⁰ operations, TLE ngay.
Solution 2 — Sort + Two Pointers
Time: O(n log n) (sort) + O(n) (two pointer) = O(n log n)
Space: O(1) hoặc O(n) tuỳ sort implementation
⚠️ Vấn đề: Bài yêu cầu return index — sau khi sort, index bị mất. Solution này chỉ đúng nếu bài hỏi giá trị, không phải index.
Solution 3 — HashMap ✅ Best
Time: O(n) — duyệt 1 lần, mỗi bước O(1) lookup.
Space: O(n) — map lưu tối đa n entries.
Đây là optimal và đúng nhất — giữ được index gốc.
Interview answer: "I'll use a HashMap to trade O(n) space for O(n) time improvement from O(n²). For each element, I store its complement's index. This is O(n) time and O(n) space."
✅ Revision Checklist
  • Tôi giải thích được O(n log n) không dùng từ "logarithm"
  • Tôi biết tại sao String += trong loop là O(n²)
  • Tôi phân biệt được O(n*m) và O(n²)
  • Tôi nhớ Arrays.sort()=O(n log n), HashMap.get()=O(1), List.contains()=O(n)
  • Tôi biết recursion depth ảnh hưởng space như thế nào

BÀI 1Group Anagrams — #49
MediumGrouping
▼
Đề bài
💡 Gợi ý
✅ Đáp án & Giải thích
Đề bài
Cho mảng strings, nhóm các anagram lại với nhau. Anagram là hai chuỗi chứa cùng các ký tự nhưng thứ tự khác.
Input
["eat","tea","tan","ate","nat","bat"]
Output
[["bat"],["nat","tan"],["ate","eat","tea"]]
Constraints: 1 ≤ strs.length ≤ 10⁴, 0 ≤ strs[i].length ≤ 100, strs[i] chỉ có lowercase letters
  1. Key insight: Hai anagram khi sort sẽ ra cùng một chuỗi. "eat" → "aet", "tea" → "aet". Dùng string đã sort làm key trong HashMap.
  2. Với mỗi string: sort nó → đó là canonical key. Thêm string gốc vào list tương ứng với key đó.
  3. Return tất cả values của map.
  4. Độ phức tạp: n strings, mỗi string dài tối đa k → sort mỗi string O(k log k) → tổng O(n × k log k)
Tư duy chính: Tìm "canonical form" — dạng chuẩn mà mọi anagram đều có chung. Sorted string là canonical form đơn giản nhất. 
public List<List<String>> groupAnagrams(String[] strs) {
    // key = sorted string, value = list of original strings
    Map<String, List<String>> groups = new HashMap<>();

    for (String s : strs) {
        // Sort characters to get canonical form
        char[] chars = s.toCharArray();
        Arrays.sort(chars);
        String key = new String(chars); // "eat" → "aet"

        // computeIfAbsent: create list if key doesn't exist
        groups.computeIfAbsent(key, k -> new ArrayList<>()).add(s);
    }

    return new ArrayList<>(groups.values());
}
TimeO(n × k log k)
SpaceO(n × k)
Dry Run với ["eat","tea","tan"]
"eat" → sort → "aet" → groups = {"aet": ["eat"]}
"tea" → sort → "aet" → groups = {"aet": ["eat","tea"]}
"tan" → sort → "ant" → groups = {"aet": ["eat","tea"], "ant": ["tan"]}
return [["eat","tea"], ["tan"]]
Alternative key (faster): Thay vì sort, dùng int[26] count array và convert sang string "1#0#0#..." — O(k) thay O(k log k).
BÀI 2Subarray Sum Equals K — #560
MediumPrefix Sum
▼
Đề bài
💡 Gợi ý
✅ Đáp án & Giải thích
Đề bài
Cho mảng số nguyên nums và số nguyên k, trả về số lượng subarray liên tiếp có tổng bằng k.
Input
nums = [1,2,3], k = 3
Output
2
([1,2] và [3])
  1. Brute force O(n²): Với mỗi cặp (i,j), tính tổng nums[i..j] và check = k. Có thể viết được ngay — nhưng TLE với n lớn.
  2. Insight: sum(i..j) = prefixSum[j+1] - prefixSum[i]. Nếu sum(i..j) = k, thì prefixSum[j+1] - prefixSum[i] = k, tức là prefixSum[i] = prefixSum[j+1] - k.
  3. Duyệt từng j, tính prefixSum[j+1], rồi hỏi: có bao nhiêu i trước đó mà prefixSum[i] = prefixSum[j+1] - k? → Dùng HashMap lưu count của mỗi prefixSum đã gặp.
  4. Base case quan trọng: Khởi tạo map với {0: 1} — nghĩa là "empty prefix" có sum = 0 đã xuất hiện 1 lần, cho phép đếm subarray bắt từ đầu mảng.
Core insight: sum[i..j] = prefixSum[j] - prefixSum[i-1]. Ta cần đếm số cặp (i,j) sao cho hiệu = k. HashMap cho phép lookup O(1) thay vì O(n) vòng lặp. 
public int subarraySum(int[] nums, int k) {
    Map<Integer, Integer> prefixCount = new HashMap<>();
    prefixCount.put(0, 1); // base case: empty prefix

    int currentSum = 0, count = 0;

    for (int num : nums) {
        currentSum += num;

        // How many previous prefixes had sum = (currentSum - k)?
        // Each such prefix creates a valid subarray ending here
        count += prefixCount.getOrDefault(currentSum - k, 0);

        // Record current prefix sum
        prefixCount.put(currentSum,
            prefixCount.getOrDefault(currentSum, 0) + 1);
    }
    return count;
}
TimeO(n)
SpaceO(n)
Dry Run — nums=[1,2,3], k=3
map={0:1}, sum=0, count=0
num=1: sum=1, need=1-3=-2, map[-2]=0 → count=0. map={0:1,1:1}
num=2: sum=3, need=3-3=0, map[0]=1 → count=1. map={0:1,1:1,3:1}
num=3: sum=6, need=6-3=3, map[3]=1 → count=2. Done ✓
✅ Revision Checklist
  • Tôi biết khi nào dùng HashMap vs HashSet
  • Tôi nhớ getOrDefault() và computeIfAbsent()
  • Tôi hiểu tại sao prefix sum cần khởi tạo {0: 1}
  • Tôi giải được Group Anagrams dùng sorted string làm key
  • Tôi viết được Subarray Sum = K không cần nhìn code

public List<List<Integer>> threeSum(int[] nums) {
    Arrays.sort(nums); // MUST sort first
    List<List<Integer>> result = new ArrayList<>();

    for (int i = 0; i < nums.length - 2; i++) {
        // Skip duplicate first elements
        if (i > 0 && nums[i] == nums[i - 1]) continue;

        int left = i + 1, right = nums.length - 1;
        while (left < right) {
            int sum = nums[i] + nums[left] + nums[right];
            if (sum == 0) {
                result.add(Arrays.asList(nums[i], nums[left], nums[right]));
                // Skip duplicates after finding a triplet
                while (left < right && nums[left] == nums[left+1]) left++;
                while (left < right && nums[right] == nums[right-1]) right--;
                left++; right--;
            } else if (sum < 0) {
                left++;  // need larger sum
            } else {
                right--; // need smaller sum
            }
        }
    }
    return result;
}

public int trap(int[] height) {
    int left = 0, right = height.length - 1;
    int maxLeft = 0, maxRight = 0;
    int water = 0;

    while (left < right) {
        if (height[left] < height[right]) {
            // Right is taller → water at left depends on maxLeft
            if (height[left] >= maxLeft) {
                maxLeft = height[left]; // update max, no water
            } else {
                water += maxLeft - height[left]; // trap water
            }
            left++;
        } else {
            // Left is taller or equal → water at right depends on maxRight
            if (height[right] >= maxRight) {
                maxRight = height[right];
            } else {
                water += maxRight - height[right];
            }
            right--;
        }
    }
    return water;
}

SpaceO(1)
Why greedy works: Khi height[left] < height[right], ta biết right sẽ không là bottleneck cho left. Dù maxRight chưa chắc là max thực sự của right side, nó luôn ≥ height[right] > height[left], nên maxLeft là giới hạn của left. Đây là greedy choice an toàn.

    while (isInvalid(window)) {

public int lengthOfLongestSubstring(String s) {
    Set<Character> window = new HashSet<>();
    int left = 0, maxLen = 0;

    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);
        // Shrink window until c is not in window
        while (window.contains(c)) {
            window.remove(s.charAt(left++));
        }
        window.add(c);
        maxLen = Math.max(maxLen, right - left + 1);
    }
    return maxLen;
}

public int lengthOfLongestSubstringOpt(String s) {
    Map<Character, Integer> lastIndex = new HashMap<>();
    int left = 0, maxLen = 0;
    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);
        if (lastIndex.containsKey(c) && lastIndex.get(c) >= left)
            left = lastIndex.get(c) + 1; // jump left past duplicate
        lastIndex.put(c, right);
        maxLen = Math.max(maxLen, right - left + 1);
    }
    return maxLen;
}

public int minEatingSpeed(int[] piles, int h) {
    int left = 1;
    int right = Arrays.stream(piles).max().getAsInt();

    while (left < right) {
        int mid = left + (right - left) / 2;
        if (canFinish(piles, mid, h)) {
            right = mid;     // mid works, try smaller
        } else {
            left = mid + 1; // mid too slow, need faster
        }
    }
    return left; // left == right == minimum valid speed
}

private boolean canFinish(int[] piles, int speed, int h) {
    long hours = 0;
    for (int pile : piles)
        hours += (pile + speed - 1) / speed; // ceil division
    return hours <= h;
}

public void reorderList(ListNode head) {
    if (head == null || head.next == null) return;

    // Step 1: Find middle using fast/slow
    ListNode slow = head, fast = head;
    while (fast.next != null && fast.next.next != null) {
        slow = slow.next;
        fast = fast.next.next;
    }

    // Step 2: Reverse second half
    ListNode secondHalf = reverse(slow.next);
    slow.next = null; // cut the list

    // Step 3: Merge two halves
    ListNode first = head, second = secondHalf;
    while (second != null) {
        ListNode tmp1 = first.next, tmp2 = second.next;
        first.next = second;
        second.next = tmp1;
        first = tmp1;
        second = tmp2;
    }
}

private ListNode reverse(ListNode head) {
    ListNode prev = null, curr = head;
    while (curr != null) {
        ListNode next = curr.next;
        curr.next = prev;
        prev = curr; curr = next;
    }
    return prev;
}

// Returns max SINGLE path gain from this node upward
private int dfs(TreeNode node) {
    if (node == null) return 0;

    // Only take positive contributions
    int leftGain  = Math.max(dfs(node.left),  0);
    int rightGain = Math.max(dfs(node.right), 0);

    // This node as bend point: use both sides
    int pathThroughNode = node.val + leftGain + rightGain;
    maxSum = Math.max(maxSum, pathThroughNode);

    // Return single path (can't bend) for parent
    return node.val + Math.max(leftGain, rightGain);
}

class MedianFinder {
    // maxHeap: lower half (top = max of lower)
    private PriorityQueue<Integer> lower = new
        PriorityQueue<>(Collections.reverseOrder());
    // minHeap: upper half (top = min of upper)
    private PriorityQueue<Integer> upper = new PriorityQueue<>();

    public void addNum(int num) {
        lower.offer(num); // add to lower first
        upper.offer(lower.poll()); // balance: push max of lower to upper

        // Keep lower.size() >= upper.size()
        if (upper.size() > lower.size())
            lower.offer(upper.poll());
    }

    public double findMedian() {
        if (lower.size() > upper.size())
            return lower.peek(); // odd total
        return (lower.peek() + upper.peek()) / 2.0; // even total
    }
}

Input
candidates=[2,3,6,7], target=7
Output
[[2,2,3],[7]]
  1. Dùng backtracking template. Vì được reuse phần tử, next iteration bắt đầu từ i (không phải i+1).
  2. Pruning: Nếu candidates[i] > remaining target, dừng luôn (nếu đã sort candidates).
  3. Base case: remaining = 0 → add current combination to result.
public List<List<Integer>> combinationSum(int[] candidates, int target) {
    List<List<Integer>> result = new ArrayList<>();
    Arrays.sort(candidates); // for pruning
    backtrack(candidates, target, 0, new ArrayList<>(), result);
    return result;
}

private void backtrack(int[] c, int remaining, int start,
                       List<Integer> current, List<List<Integer>> result) {
    if (remaining == 0) {
        result.add(new ArrayList<>(current)); // copy!
        return;
    }
    for (int i = start; i < c.length; i++) {
        if (c[i] > remaining) break; // pruning: sorted, no need to continue
        current.add(c[i]);
        backtrack(c, remaining - c[i], i, current, result); // i not i+1 (reuse)
        current.remove(current.size() - 1);
    }
}
i vs i+1: Combination Sum dùng i (reuse). Subsets dùng i+1 (no reuse). Permutations dùng visited[] (order matters).

Có numCourses khóa học (0 đến n-1). prerequisites[i] = [a, b] nghĩa là phải học b trước a. Trả về một thứ tự học hợp lệ. Nếu không thể → return [].
Input
n=4, pre=[[1,0],[2,0],[3,1],[3,2]]
Output
[0,1,2,3] hoặc [0,2,1,3]
  1. Đây là Topological Sort — sắp xếp nodes sao cho mọi edge a→b đều có a trước b.
  2. Dùng Kahn's algorithm (BFS): Tính in-degree (số prerequisite) của mỗi node. Thêm tất cả node có in-degree = 0 vào queue.
  3. Dequeue một node, thêm vào result. Giảm in-degree của tất cả neighbor. Nếu neighbor có in-degree = 0 → thêm vào queue.
  4. Nếu result.size() = n → valid order. Nếu không → có cycle → return [].
public int[] findOrder(int n, int[][] prereqs) {
    // Build adjacency list and in-degree count
    List<List<Integer>> adj = new ArrayList<>();
    int[] inDegree = new int[n];
    for (int i = 0; i < n; i++) adj.add(new ArrayList<>());
    for (int[] p : prereqs) {
        adj.get(p[1]).add(p[0]); // b → a
        inDegree[p[0]]++;
    }

    // BFS from nodes with no prerequisites
    Queue<Integer> q = new LinkedList<>();
    for (int i = 0; i < n; i++)
        if (inDegree[i] == 0) q.offer(i);

    int[] order = new int[n];
    int idx = 0;
    while (!q.isEmpty()) {
        int course = q.poll();
        order[idx++] = course;
        for (int next : adj.get(course)) {
            if (--inDegree[next] == 0) q.offer(next);
        }
    }
    return idx == n ? order : new int[0]; // cycle check
}

SpaceO(V+E)

Input

Output

  1. State: dp[i] = số đồng xu tối thiểu để đổi ra i đồng. Ta cần dp[amount].
  2. Transition: dp[i] = min(dp[i - coin] + 1) với mọi coin ≤ i. Nếu dùng coin này, ta cần dp[i - coin] cách đổi phần còn lại.
  3. Base case: dp[0] = 0 (không cần đồng nào để đổi 0). dp[1..amount] = Infinity ban đầu.
  4. Final: nếu dp[amount] vẫn là Infinity → return -1.
DP state visualization: dp[i] = "cần bao nhiêu đồng để tạo ra đúng i xu?" 
public int coinChange(int[] coins, int amount) {
    int[] dp = new int[amount + 1];
    Arrays.fill(dp, amount + 1); // "Infinity" = amount+1 (impossible)
    dp[0] = 0; // base case

    for (int i = 1; i <= amount; i++) {
        for (int coin : coins) {
            if (coin <= i) {
                // Option: use this coin, then need dp[i-coin] more
                dp[i] = Math.min(dp[i], dp[i - coin] + 1);
            }
        }
    }

    return dp[amount] > amount ? -1 : dp[amount];
}

SpaceO(amount)

Cho 2 chuỗi text1, text2. Tìm độ dài Longest Common Subsequence (LCS) — chuỗi con chung dài nhất (không cần liên tiếp).
Input
text1="abcde", text2="ace"
Output
3 ("ace")
  1. State: dp[i][j] = LCS của text1[0..i-1] và text2[0..j-1].
  2. Transition: Nếu text1[i-1] == text2[j-1] → dp[i][j] = dp[i-1][j-1] + 1 (ký tự này được chọn vào LCS). Nếu không → dp[i][j] = max(dp[i-1][j], dp[i][j-1]).
  3. Base case: dp[0][j] = dp[i][0] = 0 (empty string → LCS = 0).
public int longestCommonSubsequence(String text1, String text2) {
    int m = text1.length(), n = text2.length();
    int[][] dp = new int[m + 1][n + 1];
    // dp[0][...] = dp[...][0] = 0 (auto initialized)

    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (text1.charAt(i-1) == text2.charAt(j-1)) {
                dp[i][j] = dp[i-1][j-1] + 1; // match!
            } else {
                dp[i][j] = Math.max(dp[i-1][j], dp[i][j-1]); // skip one
            }
        }
    }
    return dp[m][n];
}
TimeO(m×n)
SpaceO(m×n)
Space optimization: Chỉ cần 2 rows thay vì m×n matrix vì dp[i][j] chỉ phụ thuộc dp[i-1][*]. → Space O(n).
